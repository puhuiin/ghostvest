## Q&A 预案

以下是评委/观众可能提出的问题及参考回答：

### Q1：HMAC-SHA256 是不是比 ZK 方案隐私性差？

> **回答**：两者隐私模型不同。ZK 提供数学上的零知识证明，但代价是高 Gas、高延迟、复杂实现。HMAC 确定性派生提供了**计算上不可逆**的隐私保护 — 给定隐身地址，无法反推原始公钥（在不知道 nonce 的前提下）。对于 DAO 薪酬场景，这种隐私强度足够，而且性能和兼容性优势明显。

### Q2：如果 nonce 泄露了怎么办？

> **回答**：nonce 通过 0G 网络加密存储，不会在链上暴露。即使 0G 不可用，也只是降级到本地 fallback.json，不会写入任何公共数据。在 v1.0 规划中，我们计划实现 ZK 增强层，为 nonce 本身提供额外的零知识保护。

### Q3：AI 审核的准确性如何保证？

> **回答**：当前使用 Mock 模式演示。在生产环境中，Claude 在 TEE 内运行，审核逻辑不可篡改。我们设定了 score ≥ 0.7 的阈值，低于阈值的 PR 会拒绝发放。此外，审核结果会和 nonce 一起存入 0G 网络作为可审计的存证记录。

### Q4：为什么选择 Solana 而不是 EVM？

> **回答**：Solana 的 ed25519 密钥系统天然适合 HMAC-SHA256 派生 — 32 字节种子直接映射到 ed25519_from_seed，数学上简洁优雅。EVM 使用 secp256k1，需要额外的曲线运算。v1.0 计划增加 EVM 多链支持。

### Q5：目前有哪些真实的用户或合作伙伴？

> **回答**：GhostVest 是在 Colosseum Hackathon 2026 中开发的 MVP。v0.1.0 的目标是验证技术可行性 — 确定性隐身地址的跨语言一致性、AI 审计的集成流程、0G 存证的降级策略。v0.2.0 将接入真实 API 和 0G SDK，v1.0 计划上主网。
